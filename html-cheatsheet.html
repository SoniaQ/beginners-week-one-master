Quick Overview

HTML dictates the structure of a website through the use of HTML tags. These tags describe the various elements of an HTML document.

DOM - Document Object Model

The DOM is the specified hierarchical layout of the html document i.e. skeleton (also known as boiler plate). It always begins with !DOCTYPE html, which tells you what version of HTML is being used (html5, html4...). HTML5 is the latest version, described simply as html.

The HTML tag defines the route of a document and everything, except !DOCTYPE, is wrapped whithin these.
Code between the HEAD tags provide information about the page and includes tags such as: TITLE, META, etc. For example, text within the TITLE tags will be displayed in the browser bar.
Items (text, images, links, videos, etc) to be displayed on the webpage are enclosed within the BODY tags.

Headings and paragraphs (this is an h2 heading)

Headings are marked using heading tags such as h1. There are six levels of heading supported. The biggest is h1 (used for page titles). The smallest is h6. On most pages you probably won't see h4-6 used.

Paragraphs are separated using p tags. This automatically separates them into blocks, leaving a gap in between.

Blank lines and void tags

To insert extra blank lines manually the br tag is used. It is a special type of tag (called a 'void tag'); it is a self-closing tag.


Comments

You can write comments in your html code, which won't be displayed in the browser.

Auto indent

In Atom, navigate to Edit - Lines - Auto indent.

Lists

Html has two basic types of list:

  1. Ordered lists (<ol>) - displays a numbered list (1,2, 3; a, b, c; i, ii, iii; etc)
  2. Unordered lists (<ul>) - displays a list of bullet points

Both have the same basic structure.

  • An opening list tag
  • Many list items (<li>)
  • A closing list tag
  • The <li> tags are nested between the <ol> or <ul> tags, depending on the desired list type

Lists aren't often seen in their basic form, but they are heavily used in websites; dropdown menus are often cleverly styled lists.

Links

Links look like this: link to facebook. The 'href' attribute, located inside the anchor (<a>) tags, specifies where the link points to.

The href can also point to local files. For example, this is a link to this page you're currently reading. Notice that the protocol and host part of the URL has been replaced by a "."

Images

Images are similar to links. The 'src' property, located within the <img> tag, specifies the location fo the image. Note that img tags are like br tags: they don't have a closing tag. The 'alt' property should be used to provide a description of the image in case it won't load or the user is partially sighted.
